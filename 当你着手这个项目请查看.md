## 项目简介

Smart QA — 智能文档问答系统，基于 RAG（检索增强生成）架构。用户上传文档后，系统自动切片、向量化、建立索引，用户提问时通过混合检索 + Rerank 重排序找到最相关的内容，交给 LLM 生成回答并标注来源。v2.1 新增知识图谱、答案评估、文档摘要等六大功能。

**适用场景：** 毕业设计、知识库问答、文档检索

---

## 技术栈

| 组件 | 技术 | 说明 |
|------|------|------|
| 后端框架 | FastAPI | Python 异步 Web 框架 |
| 前端 | HTML/CSS/JS | 仿 DeepSeek 风格单页应用 |
| 向量数据库 | ChromaDB | 本地持久化，存储在 data/chroma/ |
| Embedding | paraphrase-multilingual-MiniLM-L12-v2 | 多语言小模型，首次运行自动下载 |
| BM25 检索 | rank-bm25 + jieba | 中文关键词检索 |
| 检索融合 | RRF（Reciprocal Rank Fusion） | 向量 + BM25 结果融合 |
| Rerank | BGE-Reranker-Base | CrossEncoder 精排（~280MB） |
| OCR | PaddleOCR | 扫描件 PDF 自动识别 |
| 文档解析 | mammoth | DOCX 完整提取（含文本框/表格） |
| 知识图谱 | NetworkX | 实体关系图谱 + GraphRAG |
| LLM | OpenAI 兼容 API | DeepSeek / Qwen / Claude 等 |

## 快速启动

```bash
# 进入项目目录
cd Desktop/smart-qa

# 安装依赖（首次）
pip install -r requirements.txt

# 终端1：启动后端（端口 8000）
python run_backend.py

# 终端2：启动前端（端口 8501）
python run_frontend.py

# 浏览器打开
http://localhost:8501
```

## 核心流程

```
用户提问
  ↓
判断是否简单问题（≤15字/问候语 → 跳过 HyDE）
  ↓
HyDE 查询改写（用 LLM 生成假想答案，用答案做检索）
  ↓
混合检索（向量检索 + BM25 关键词检索）
  ↓
RRF 融合（Reciprocal Rank Fusion 合并排名）
  ↓
CrossEncoder 重排序（BGE-Reranker 精排）
  ↓
知识图谱检索（匹配实体 → 获取关联实体和关系）
  ↓
对话压缩（超 10 轮时压缩旧历史为摘要）
  ↓
LLM 生成回答（带来源引用 + 图谱上下文）
  ↓
流式返回前端 + 引用高亮 + 异步质量评估 + 自动保存会话
```

## API 接口

|------|------|------|

## 关键配置（.env 文件）

```env
# LLM API
LLM_API_KEY=你的 API Key
LLM_BASE_URL=https://api.deepseek.com/v1
LLM_MODEL=deepseek-v4-flash

# Embedding 模型（可选，不填用默认值）
# EMBEDDING_MODEL=paraphrase-multilingual-MiniLM-L12-v2

# Reranker（可选，不填用默认值）
# ENABLE_RERANK=true
# RERANKER_MODEL=BAAI/bge-reranker-base
# RERANK_TOP_N=5

# 切片策略（可选）
# CHUNK_SIZE=500
# CHUNK_STRATEGY=fixed   # fixed（默认）或 semantic（语义切片）
```

所有配置项在 `backend/config.py` 中有默认值，可通过 .env 覆盖。

## 注意事项

1. **API Key 安全：** `.env` 已在 `.gitignore` 中，切勿提交到 Git
2. **首次运行：** 会自动下载 Embedding 模型和 Reranker 模型，需要联网
3. **OCR：** 扫描件 PDF 首次 OCR 会自动下载 PaddleOCR 模型（约 100MB）
4. **ChromaDB 数据：** 存储在 `data/chroma/`，删除此目录可清空向量库（需先停后端）
5. **知识图谱数据：** 存储在 `data/graph/knowledge_graph.json`，删除可清空图谱
6. **BM25 索引：** 每次后端启动时从 ChromaDB 自动重建，无需手动维护
7. **日志：** 运行日志在 `data/logs/smartqa.log`，方便调试
8. **端口：** 后端 8000，前端 8501，如需修改编辑 run_backend.py 和 run_frontend.py
9. **语义切片：** 需要先计算 embedding，首次使用会稍慢，适合对切片质量要求高的场景
10. **知识图谱：** 上传时自动抽取实体关系，会增加一次 LLM 调用耗时
