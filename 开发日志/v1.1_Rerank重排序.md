# v1.1 Rerank 重排序

**日期：** 2026年5月3日

## 概述

在混合检索后增加 CrossEncoder 重排序步骤，显著提升检索结果质量。

## 问题

原有流程：用户问题 → HyDE改写 → 向量检索+BM25 → RRF融合 → 返回top_k

RRF 融合只考虑了各检索器的排名，没有对 (query, document) 的实际相关性做精排。当候选文档较多时，可能遗漏最相关的文档。

## 方案

引入 BGE-Reranker（CrossEncoder 模型），对 RRF 融合后的候选文档做二次排序。

新流程：用户问题 → HyDE改写 → 向量检索+BM25 → RRF融合 → **CrossEncoder重排序** → 返回top_k

## 改动

### 新增文件
- `backend/reranker.py` — CrossEncoder 重排序模块
  - 使用 `sentence-transformers.CrossEncoder`
  - 默认模型：`BAAI/bge-reranker-base`（~280MB，支持中文）
  - 懒加载：首次调用时下载并加载模型
  - 输入：(query, doc_content) 对 → 输出：相关性分数

### 修改文件
- `backend/config.py` — 新增配置项：
  - `enable_rerank: bool = True` — 是否启用重排序
  - `reranker_model: str = "BAAI/bge-reranker-base"` — Reranker 模型名
  - `rerank_top_n: int = 5` — 重排序后返回数量

- `backend/vector_store.py` — `hybrid_search()` 集成 rerank：
  - 导入 `reranker` 模块
  - 当 `enable_rerank=True` 时，对 RRF 结果调用 `rerank()`

- `.env.example` — 添加 reranker 配置示例

## 检索流程对比

```
v1.0: query → HyDE → [向量, BM25] → RRF → top_5
v1.1: query → HyDE → [向量, BM25] → RRF → CrossEncoder rerank → top_5
```

## 配置

```env
# .env 中可配置
ENABLE_RERANK=true
RERANKER_MODEL=BAAI/bge-reranker-base
RERANK_TOP_N=5
```

设置 `ENABLE_RERANK=false` 可关闭重排序，回退到纯 RRF 融合。
