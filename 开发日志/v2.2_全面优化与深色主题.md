# v2.2 全面优化与深色主题

**日期：** 2026年5月9日

## 概述

对前端 UI、后端性能、代码结构进行全面优化。前端升级为统一深色主题，解决流式输出卡顿问题；后端实现检索并行化和路由拆分；升级 Embedding 模型为中文优化版本，显著提升检索质量。

---

## 一、前端优化

### 1.1 流式渲染性能优化

**问题：** 每收到一个 SSE chunk 都对全文做 `marked.parse()` 并替换 `innerHTML`，文本越长越卡。

**修复：** 改用 `requestAnimationFrame` 节流，DOM 更新频率从每 chunk 一次降到每帧一次。`scrollToBottom` 同样加入 rAF 节流。

### 1.2 深色主题

**改动：** 全站切换为统一深色主题（主背景 `#12121a`），替换之前的浅色主区域 + 深色侧边栏割裂风格。

主要变更：
- CSS 变量全部重写为深色色板
- highlight.js 主题从 `github.min` 切换到 `github-dark.min`
- 消息气泡、卡片、边框、输入框等统一深色调
- Logo 增加渐变光晕效果，标题使用渐变文字

### 1.3 停止生成与重新生成

- 输入区新增红色停止按钮（发送时自动切换），支持 `AbortController` 中断流式请求
- 每条 AI 回答下方新增"重新生成"按钮 + 反馈按钮组合
- 重新生成显示生成次数标记（如 "2/2"、"3/3"）
- 停止生成时保留已输出内容，显示"已停止生成"提示

### 1.4 代码块复制

- AI 回答中的代码块自动包裹 `.code-block-wrapper`
- hover 时显示"复制"按钮，点击复制代码到剪贴板
- 复制成功后显示"已复制"反馈，2 秒后恢复

### 1.5 其他 UI 改进

- 消息气泡下方显示发送时间戳
- 文件上传增加动画进度条（替代纯文字提示）
- 拖拽上传使用 CSS class 控制样式（替代 inline style）
- 消息出现动画从 `fadeIn` 升级为 `messageIn`（更细腻的弹入效果）
- 输入框 focus 时增加蓝色光晕（`box-shadow`）

## 二、后端优化

### 2.1 检索并行化

**改动：** `hybrid_search()` 中 vector_search 和 BM25 search 改为 `ThreadPoolExecutor` 并发执行。

```python
with ThreadPoolExecutor(max_workers=2) as executor:
    future_vector = executor.submit(vector_search, query, fetch_k)
    future_bm25 = executor.submit(bm25_search.search, query, fetch_k)
```

预期减少 30-50% 检索延迟。

### 2.2 路由拆分

**改动：** `main.py` 从 504 行拆分为 4 个路由模块：

| 模块 | 职责 |
|------|------|
| `routes/documents.py` | 文档上传/列表/删除/摘要 |
| `routes/chat.py` | 聊天/评估/反馈 + HyDE 逻辑 |
| `routes/sessions.py` | 会话 CRUD |
| `routes/knowledge.py` | 知识库统计/片段浏览 |

`main.py` 精简为 70 行，只负责 app 初始化和路由注册。

### 2.3 切片增加 Overlap

**改动：** `text_splitter.py` 新增 `chunk_overlap` 配置（默认 50 字符），每个 chunk 前面附加上一个 chunk 的尾部内容，保持语义连贯。

### 2.4 System Prompt 优化

**改动：** 优化 LLM system prompt，从严格限制"根据资料回答"改为"资料作为补充，相关性不高时结合自身知识回答"，避免不相关资料干扰模型判断。

## 三、模型升级

### 3.1 Embedding 模型

| 项目 | 之前 | 之后 |
|------|------|------|
| 模型 | `paraphrase-multilingual-MiniLM-L12-v2` | `BAAI/bge-small-zh-v1.5` |
| 参数量 | ~22M | ~90M |
| 维度 | 384 | 512 |
| 优化方向 | 多语言通用 | 中文检索专用 |

中文语义理解能力显著提升，检索召回更准确。

### 3.2 注意事项

- 切换模型后需清空 `data/chroma/` 或更换集合名，旧向量数据不兼容
- 新模型首次启动会自动下载（约 90MB）
