# v2.1 知识图谱功能移除

**日期：** 2026年5月5日

## 概述

尝试为知识图谱可视化添加交互功能（拖拽、缩放、悬停），发现 canvas 在模态框内无法接收鼠标事件，经多次调试未解决。最终决定移除整个知识图谱模块，精简代码。

---

## 一、交互式图谱尝试（已回滚）

### 目标

将知识图谱从 120 帧静态动画升级为可交互的力导向图：

- 鼠标拖拽节点
- 滚轮缩放画布
- 右键平移
- 悬停高亮关联边 + tooltip 显示实体详情

### 问题

Canvas 在模态框（`.modal-overlay`）内完全无法接收鼠标事件。`mousedown` 事件监听器不触发，控制台无任何日志输出。排查过程：

1. 检查 canvas 尺寸 — `getBoundingClientRect()` 返回正常宽高
2. 检查节点坐标 — 初始化位置正确
3. 检查 `hitTest` 逻辑 — 无逻辑错误
4. 添加 `console.log` 调试 — mousedown 根本不触发
5. 怀疑是模态框 CSS 层级/事件捕获机制阻止了 canvas 的事件传播

**结论：** canvas 在动态创建的模态框内存在事件传播问题，根因未定位。如需交互式图谱，建议改用 SVG 或独立页面方案。

---

## 二、知识图谱功能完整移除

决定移除整个知识图谱模块，包括前后端所有相关代码。

### 2.1 前端删除

**app.js：**
- DOM 引用：`graphStatsRow`、`graphStatsValue`、`graphModal`、`graphModalClose`、`graphBody`
- `fetchStats()` 中的图谱统计请求
- `GRAPH_COLORS` 常量
- `openGraphModal()`、`loadGraph()`、`renderGraph()` 函数
- 图谱模态框的事件监听

**index.html：**
- 侧边栏「知识图谱」统计行（`#graphStatsRow`）
- 图谱弹窗（`#graphModal`）

**style.css：**
- `.graph-container`、`.graph-canvas`、`.graph-legend`、`.graph-stats-bar` 等样式

### 2.2 后端删除

**knowledge_graph.py：** 整个文件删除

包含：NetworkX 图存储、LLM 实体关系抽取、图谱搜索、图谱上下文注入、按来源删除等功能。

**main.py：**
- 移除 `from backend.knowledge_graph import ...`
- 移除上传文档时的 `extract_from_chunks()` 调用
- 移除聊天时的 `graph_search_context()` 调用及 `graph_context` 参数传递
- 移除 `GET /graph` 和 `GET /graph/data` 两个接口
- 移除删除文档时的 `kg_delete_by_source()` 调用

**llm.py：**
- `chat()` 和 `chat_stream()` 移除 `graph_context` 参数及相关的上下文拼接逻辑

## 三、影响范围

| 变更 | 影响 |
|------|------|
| 上传文档 | 不再进行实体关系抽取，上传速度更快 |
| 聊天问答 | 不再注入图谱上下文，仅依赖向量+BM25混合检索 |
| 文档删除 | 不再清理图谱数据（已无图谱） |
| 前端界面 | 侧边栏少一行「知识图谱」统计，无弹窗 |
| 依赖 | NetworkX 仍在 requirements.txt 中但不再使用 |
